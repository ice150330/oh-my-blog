### 2.1 推荐配置

| 项目 | 规格 | 说明 |
|------|------|------|
| 服务器 | 腾讯云/阿里云 轻量应用服务器 2核4G | 国内访问最快 |

| 域名 | `.com` 或 `.cn` | 任意注册商 |
| **首年费用** | **¥134-375** | 新用户活动价 |

### 2.2 服务器初始化

```bash
# SSH 登录
ssh root@YOUR_SERVER_IP

# 创建部署用户（不要用 root 运行应用）
useradd -m -s /bin/bash deploy
usermod -aG sudo deploy
passwd deploy

# 系统更新 + 基础工具
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl wget vim git htop nginx certbot python3-certbot-nginx

# 安装 Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker deploy
# 重新登录使 docker 组生效

# 配置 Swap（4GB 兜底）
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 配置防火墙（仅开放必要端口）
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw enable

# 验证
docker --version
free -h | grep Swap
```

## 三、ICP 备案流程

### 3.1 备案时间线

| 步骤 | 操作 | 耗时 | 说明 |
|------|------|------|------|
| Day 1 | 购买服务器 + 域名 | 30 min | 域名需实名认证（0-24h） |
| Day 1 | 通过服务商 App 提交备案 | 30 min | 身份证 + 人脸核验 + 承诺书 |
| Day 1-12 | **备案审核中** | 7-12 天 | 腾讯云/阿里云平均审核时间 |
| Day 7-20 | **管局审核** | 不等 | 短信核验 |
| 审核通过 | 域名解析生效 | 即时 | 添加 A 记录指向服务器 IP |

> **备案期间不影响开发**：可通过服务器 IP 直接访问验证部署，或用本地 hosts 绑定。

### 3.2 加速技巧

- 选腾讯云/阿里云（比小服务商审核快 3-5 天）
- 提前准备：身份证正反面、白底半身照
- 避开节假日提交
- 已有备案号的走"接入备案"仅需 3-5 天

## 四、Docker 生产部署

### 4.1 docker-compose.yml（生产）

```yaml
version: '3.8'
services:
  # Spring Boot 应用
  app:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: portfolio-app
    ports:
      - "127.0.0.1:8080:8080"  # 仅本机暴露
    environment:
      - JAVA_OPTS=-Xms512m -Xmx512m -XX:+UseG1GC -XX:MaxGCPauseMillis=200
      - SPRING_PROFILES_ACTIVE=prod
      - MYSQL_USER=${MYSQL_USER}
      - MYSQL_PASSWORD=${MYSQL_PASSWORD}
      - JWT_SECRET=${JWT_SECRET}
    deploy:
      resources:
        limits:
          memory: 1G
        reservations:
          memory: 512M
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_started
    restart: unless-stopped
    logging:
      driver: "json-file"
      options:
        max-size: "10m"
        max-file: "3"

  # MySQL
  mysql:
    image: mysql:8.0
    container_name: portfolio-mysql
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: portfolio
      MYSQL_USER: ${MYSQL_USER}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD}
    command: >
      --default-authentication-plugin=mysql_native_password
      --innodb-buffer-pool-size=256M
      --max-connections=50
    volumes:
      - mysql_data:/var/lib/mysql
      - ./backend/src/main/resources/init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    deploy:
      resources:
        limits:
          memory: 512M
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 3

  # Redis
  redis:
    image: redis:7-alpine
    container_name: portfolio-redis
    command: redis-server --maxmemory 128mb --maxmemory-policy allkeys-lru --appendonly no
    deploy:
      resources:
        limits:
          memory: 256M
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 3s
      retries: 3

volumes:
  mysql_data:
```

## 五、Nginx 配置

### 5.1 主配置

```nginx
# /etc/nginx/sites-available/portfolio

# HTTP → HTTPS 重定向
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

# HTTPS 主站
server {
    listen 443 ssl http2;
    server_name yourdomain.com www.yourdomain.com;

    # SSL 证书（certbot 自动管理）
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 前端静态文件
    root /var/www/frontend/dist;
    index index.html;

    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/javascript application/json image/svg+xml;
    gzip_comp_level 6;
    gzip_min_length 256;

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    # 前端 SPA 路由
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 项目 Demo 静态文件
    location /demos/ {
        alias /var/www/demos/;
        try_files $uri $uri/ =404;
        add_header Cache-Control "public, max-age=86400";
    }

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 10s;
        proxy_read_timeout 30s;
    }

    # SSE 长连接代理（禁用缓冲）
    location /sse/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_buffering off;
        proxy_read_timeout 86400s;
        proxy_connect_timeout 10s;
    }

    # 静态资源缓存（6 个月）
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 6M;
        add_header Cache-Control "public, immutable";
        access_log off;
    }
}
```

### 5.2 启用站点

```bash
sudo ln -s /etc/nginx/sites-available/portfolio /etc/nginx/sites-enabled/
sudo nginx -t          # 测试配置
sudo systemctl reload nginx
```

## 八、备份策略

### 8.1 数据库备份脚本

```bash
#!/bin/bash
# /opt/scripts/backup-db.sh

BACKUP_DIR="/opt/backups/mysql"
RETENTION_DAYS=7
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

docker exec portfolio-mysql mysqldump \
  -u root -p${MYSQL_ROOT_PASSWORD} \
  --single-transaction \
  --routines --triggers \
  portfolio | gzip > "${BACKUP_DIR}/portfolio_${TIMESTAMP}.sql.gz"

# 删除 7 天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +${RETENTION_DAYS} -delete

echo "Backup completed: portfolio_${TIMESTAMP}.sql.gz"
```

### 8.2 Crontab 定时任务

```bash
# 每天凌晨 3 点备份数据库
0 3 * * * /opt/scripts/backup-db.sh >> /var/log/backup.log 2>&1

# 每周日凌晨 4 点清理 Docker 日志
0 4 * * 0 docker system prune -f --filter "until=168h"
```

## 十三、安全加固

1. **SSH 密钥登录**：禁用密码登录，仅允许 SSH Key
2. **fail2ban**：防止暴力破解
3. **安全组**：仅开放 22/80/443 端口，8080/3306/6379 仅本机
4. **Docker 非 root**：应用容器不要以 root 用户运行
5. **环境变量**：密钥通过 `.env` 注入，不硬编码
6. **定期更新**：`apt update && apt upgrade` 每月执行
7. **内容审查**：确保博客内容符合国内法规，不出现 VPN/翻墙等违规内容
