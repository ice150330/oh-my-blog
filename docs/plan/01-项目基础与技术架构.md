- [ ] 后端 `mvn spring-boot:run` 能访问 `http://localhost:8080/api/v1/health`
- [ ] 前端能通过 Nginx 代理访问后端 API（开发环境）
- [ ] CI/CD 流水线能自动执行 lint → build → test

## 二、技术选型确认

### 2.1 前端

| 技术 | 版本 | 选型理由 | 替代方案（已评估） |
|------|------|---------|-----------------|
| Nuxt 3 | ^3.11 | Vue 生态一致性、原生 SSG/SEO | Astro（Island架构更优雅但需额外学习成本） |
| Vue 3 | ^3.4 | 与所有展示项目技术栈统一 | - |
| TypeScript | ^5.3 | 类型安全 | - |
| Vite | ^5.1 | Nuxt 3 内置 | - |
| pnpm | ^9 | 比 npm 更快、节省磁盘 | npm / yarn |

> **关于 Astro 的评估**：Astro 的 Island Architecture 可以原生解决"多个独立 UI 嵌入"问题，不需要 iframe。但作者对 Vue/Nuxt 生态极度熟悉，Nuxt 3 SSG 已足够满足需求。如果 V2 发现 iframe 维护成本过高，可考虑迁移 Astro——届时只需重写布局层，Vue 组件可直接复用。

### 2.2 后端

| 技术 | 版本 | 选型理由 |
|------|------|---------|
| Java 17 | LTS | 企业级标准，与 SmartStorageSpace 一致 |
| Spring Boot | ^3.2 | 生态成熟 |
| MyBatis Plus | ^3.5 | ORM + 分页 |
| Spring Security | ^6.2 | JWT 认证 |
| Maven | ^3.9 | 构建工具 |

### 2.3 中间件

| 技术 | 版本 | 用途 | 资源限制 |
|------|------|------|---------|
| MySQL | 8.0 | 主数据库 | 512MB |
| Redis | 7-alpine | 缓存/热点 | 256MB |
| Nginx | 1.25-alpine | 反向代理 + 静态文件 | 128MB |

## 三、开发环境搭建

### 3.1 统一 Node 版本

用 Volta 或 `.nvmrc` 锁定 Node 版本：

```json
// frontend/package.json
{
  "volta": {
    "node": "18.20.0",
    "pnpm": "9.0.0"
  }
}
```

### 3.2 开发环境（Windows 纯本地）

开发环境在 Windows 本地直接运行，不依赖 Docker。需要手动安装以下服务：

| 服务 | 安装方式 | 配置要求 |
|------|---------|---------|
| **MySQL 8.0** | MySQL Installer for Windows | 端口 3306，账号 `root`，密码 `123456`，数据库 `portfolio_dev` |
| **Redis** | Memurai（Windows 兼容版） | 端口 6379，`maxmemory 64mb`，`maxmemory-policy allkeys-lru` |

MySQL 初始化：手动执行 `backend/src/main/resources/init.sql` 中的建表语句（首次启动后执行一次即可）。

### 3.3 本地启动命令（PowerShell）

```powershell
# 1. 确保 MySQL 和 Memurai 服务已启动（Windows 服务管理器）

# 2. 启动后端（新终端）
cd backend
mvn spring-boot:run -Dspring-boot.run.profiles=dev

# 3. 启动前端（新终端）
cd frontend
pnpm install
pnpm run dev
```

## 五、CI/CD 流水线

### 5.1 GitHub Actions：前端

```yaml
# .github/workflows/frontend-ci.yml
name: Frontend CI
on:
  push:
    paths: ['frontend/**']
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v3
        with: { version: 9 }
      - uses: actions/setup-node@v4
        with: { node-version: 18, cache: 'pnpm', cache-dependency-path: frontend/pnpm-lock.yaml }
      - run: pnpm install
        working-directory: frontend
      - run: pnpm run lint
        working-directory: frontend
      - run: pnpm run build
        working-directory: frontend
      - run: pnpm run test
        working-directory: frontend
      - uses: actions/upload-artifact@v4
        with:
          name: frontend-dist
          path: frontend/.output/public
```

### 5.2 GitHub Actions：后端

```yaml
# .github/workflows/backend-ci.yml
name: Backend CI
on:
  push:
    paths: ['backend/**']
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with: { java-version: 17, distribution: 'temurin' }
      - run: mvn -B verify
        working-directory: backend
```

## 七、开发细则

为保证项目长期可维护性，所有开发人员须严格遵守以下规范。

### 7.1 文件命名规范

| 文件类型 | 命名规范 | 示例 |
|----------|----------|------|
| Vue 组件 | 大驼峰（PascalCase） | `AppHeader.vue`、`HeroSection.vue` |
| TypeScript 组合式函数 | 小驼峰（camelCase） | `useTheme.ts`、`useDevice.ts` |
| TypeScript 工具模块 | 小驼峰（camelCase） | `apiClient.ts`、`markdownRenderer.ts` |
| SCSS 样式文件 | 小驼峰（camelCase） | `variables.scss`、`glassmorphism.scss` |
| 常量/配置 | 小驼峰（camelCase） | `errorCode.ts`、`tailwind.config.ts` |
| Nuxt 页面文件 | 小驼峰或中横线（遵循框架约定） | `index.vue`、`tech-stack.vue`、`[slug].vue` |

> **禁止**使用下划线命名（如 `app_header.vue`）或纯小写无分隔（如 `appheader.vue`）。组件文件必须为大驼峰，确保与模板中标签名一致。

### 7.2 代码注释规范

**所有注释必须使用中文**，方便团队协作与后期维护。

```typescript
// 正确：获取当前主题状态，优先从本地缓存读取
const theme = localStorage.getItem('theme');

// 错误：Get current theme status
const theme = localStorage.getItem('theme');
```

特殊例外：第三方库 API 文档引用、开源协议声明、标准算法名称可保留英文。

### 7.3 前端样式管理规范

前端样式采用 **"公共样式 + 组件级样式"** 双层架构，杜绝样式碎片化与重复定义。

#### 公共样式目录结构

```
styles/
├── _variables.scss        # 设计 Token（色彩、间距、阴影、字体、光影）
├── _lightTheme.scss       # 亮色模式变量覆写
├── _darkTheme.scss        # 暗色模式变量覆写
├── _mixins.scss           # SCSS 混合宏（玻璃拟态、光影效果、响应式断点）
├── _animations.scss       # 全局动画关键帧与工具类
├── _globals.scss          # 全局基础样式（html, body, 滚动条, 文本选中色）
├── _utilities.scss        # 原子化工具类（超出 Tailwind 范围的自定义类）
└── main.scss              # 入口文件：按序导入以上所有模块
```

#### 规范原则

1. **禁止硬编码色值**：所有颜色必须通过 CSS 变量引用（如 `var(--color-primary)`），禁止在组件中写死 `#2563EB` 等色值。
2. **禁止组件内重复定义通用动画**：进入动画统一使用 `_animations.scss` 中的工具类（如 `.animate-fade-in-up`）。
3. **组件级样式仅写布局与特殊覆盖**：组件 `.vue` 文件中的 `<style scoped>` 只负责该组件特有的定位、宽高、特殊交互状态，不定义颜色、字体、阴影等全局 Token。
4. **光影效果统一由 `_mixins.scss` 提供**：确保全站发光强度、模糊半径、过渡时长保持一致。

### 7.4 光影设计语言规范

本项目整体视觉风格定义为 **"光影设计语言（Light & Shadow Design Language）"**——在保留现代玻璃拟态（Glassmorphism）通透感的基础上，引入层次丰富的光影效果，营造科技、深邃、沉浸的视觉氛围。

#### 设计原则

| 原则 | 说明 |
|------|------|
| **光为引** | 用柔和光晕引导视觉焦点，强调按钮、卡片等交互元素 |
| **影为界** | 用细腻阴影划分层级，替代生硬的实体边框 |
| **玻璃为媒** | 保留磨砂玻璃通透感，作为信息承载的透明介质 |
| **微动为灵** | 所有交互伴随细微光影流动，反馈即时且优雅，不喧宾夺主 |

#### 核心光影 Token（补充至 `_variables.scss`）

```scss
:root {
  // 原有色彩变量保留...

  // 光影发光系统
  --glow-primary: 0 0 16px rgba(37, 99, 235, 0.25);
  --glow-primary-strong: 0 0 32px rgba(37, 99, 235, 0.35);
  --glow-soft: 0 4px 24px rgba(0, 0, 0, 0.06);
  --glow-inner-light: inset 0 1px 1px rgba(255, 255, 255, 0.15);

  // 渐变光影背景
  --gradient-hero:
    radial-gradient(ellipse at 20% 30%, rgba(37, 99, 235, 0.08) 0%, transparent 50%),
    radial-gradient(ellipse at 80% 70%, rgba(99, 102, 241, 0.06) 0%, transparent 50%);
  --gradient-card-shine:
    linear-gradient(135deg, rgba(255, 255, 255, 0.1) 0%, transparent 50%, rgba(255, 255, 255, 0.05) 100%);
}
```

#### 组件光影应用示例

```scss
// 按钮：默认内发光 + hover 外光晕扩散
.btn-primary {
  background: var(--color-primary);
  color: white;
  box-shadow: var(--glow-inner-light), var(--glow-soft);
  transition: all 0.3s ease;

  &:hover {
    box-shadow: var(--glow-inner-light), var(--glow-primary);
    transform: translateY(-2px);
  }

  &:active {
    transform: translateY(0);
    box-shadow: inset 0 2px 4px rgba(0, 0, 0, 0.15);
  }
}

// 卡片：玻璃基底 + 边缘高光 + hover 光晕增强
.card-glass {
  @include glass-base; // 调用 _mixins.scss 中的玻璃拟态混合宏
  position: relative;
  overflow: hidden;

  // 边缘高光：营造厚度与通透感
  &::before {
    content: '';
    position: absolute;
    inset: 0;
    border-radius: inherit;
    padding: 1px;
    background: var(--gradient-card-shine);
    -webkit-mask:
      linear-gradient(#fff 0 0) content-box,
      linear-gradient(#fff 0 0);
    mask:
      linear-gradient(#fff 0 0) content-box,
      linear-gradient(#fff 0 0);
    -webkit-mask-composite: xor;
    mask-composite: exclude;
    pointer-events: none;
  }

  &:hover {
    box-shadow: var(--shadow-lg), var(--glow-primary);
  }
}
```

#### 暗色模式特别处理

暗色模式下，光影效果转向 **"暗夜霓虹"** 风格——光晕饱和度略降、扩散范围适度扩大，避免刺眼：

```scss
.dark {
  --glow-primary: 0 0 20px rgba(37, 99, 235, 0.2);
  --glow-primary-strong: 0 0 40px rgba(37, 99, 235, 0.3);
  --gradient-hero:
    radial-gradient(ellipse at 20% 30%, rgba(37, 99, 235, 0.12) 0%, transparent 50%),
    radial-gradient(ellipse at 80% 70%, rgba(99, 102, 241, 0.08) 0%, transparent 50%);
}
```

> **性能约束**：同时生效的 `box-shadow` / `filter: drop-shadow()` 元素不得超过 10 个，避免重绘压力过大。复杂发光效果优先使用伪元素 `::before` 实现，减少主层合成开销。
