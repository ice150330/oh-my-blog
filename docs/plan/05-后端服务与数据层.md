## 三、API 接口规范

### 3.1 基础信息

- **Base URL**：`https://api.yourdomain.com/api/v1`
- **认证方式**：JWT Bearer Token（仅管理接口需要）
- **内容类型**：`application/json; charset=utf-8`

### 3.2 统一响应格式

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1714291200000
}
```

### 3.3 Result.java

```java
package com.portfolio.vo;

import lombok.Data;

@Data
public class Result<T> {
    private int code;
    private String message;
    private T data;
    private long timestamp;

    public static <T> Result<T> success(T data) {
        Result<T> r = new Result<>();
        r.code = 200;
        r.message = "success";
        r.data = data;
        r.timestamp = System.currentTimeMillis();
        return r;
    }

    public static <T> Result<T> error(int code, String message) {
        Result<T> r = new Result<>();
        r.code = code;
        r.message = message;
        r.timestamp = System.currentTimeMillis();
        return r;
    }
}
```

### 3.4 错误码定义

| 码值 | 含义 | 场景 |
|------|------|------|
| 200 | 成功 | 正常返回 |
| 400 | 参数错误 | 请求参数校验失败 |
| 401 | 未授权 | JWT 缺失或过期 |
| 403 | 禁止访问 | 权限不足 |
| 404 | 资源不存在 | 项目/文章 ID 无效 |
| 500 | 服务器错误 | 内部异常 |

### 4.5 SSE 实现

```java
// VisitorSseController.java
@RestController
@RequestMapping("/api/v1/visitor")
public class VisitorSseController {

    private final CopyOnWriteArrayList<SseEmitter> emitters = new CopyOnWriteArrayList<>();

    @GetMapping(value = "/sse", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public SseEmitter subscribe() {
        SseEmitter emitter = new SseEmitter(300_000L); // 5 分钟超时

        emitters.add(emitter);
        emitter.onCompletion(() -> emitters.remove(emitter));
        emitter.onTimeout(() -> emitters.remove(emitter));
        emitter.onError(e -> emitters.remove(emitter));

        // 发送当前在线人数
        try {
            emitter.send(SseEmitter.event()
                .name("online")
                .data(emitters.size()));
        } catch (IOException e) {
            emitters.remove(emitter);
        }

        return emitter;
    }

    @PostMapping("/heartbeat")
    public Result<Void> heartbeat(HttpServletRequest request) {
        // 更新访客活跃时间（用 Redis 记录）
        String ip = request.getRemoteAddr();
        redisTemplate.opsForValue().set("visitor:" + ip, "1", Duration.ofSeconds(30));

        // 广播最新在线人数给所有 SSE 连接
        long onlineCount = redisTemplate.keys("visitor:*").size();
        for (SseEmitter emitter : emitters) {
            try {
                emitter.send(SseEmitter.event()
                    .name("online")
                    .data(onlineCount));
            } catch (IOException e) {
                emitters.remove(emitter);
            }
        }

        return Result.success(null);
    }
}
```

> **SSE vs WebSocket 选型理由**：SSE 基于 HTTP 单向推送，足够满足"在线人数统计"需求。WebSocket 长连接在遭遇爬虫扫描时易耗尽连接池，SSE 的 HTTP 无状态特性更稳定。
